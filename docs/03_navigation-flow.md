> v1.1에 구현된 2개 화면과 모달 구조

```

│ 진입: http://localhost:8081/src/index.html       │
│                                                  │
│ ① 업체 정산 완료 내역 (목록)                     │
└────────────────────┬─────────────────────────────┘
                     │
                     │ ▸ 회사명 클릭
```

## 4. 화면 ① 업체 정산 완료 내역 — 클릭 가능 영역

| 영역 | 동작 |
|---|---|
| **기업체** (회사명) | → 정산상세로 이동 |
| **정산 완료 건수** (숫자) | → 정산상세로 이동 |
| **최근 진행내역 셀** | → 진행내역 팝업 (모달) |
| 검색 버튼 | 검색 실행 |
| 목록개수 셀렉트 | 페이지 크기 변경 (20/50/100) |

## 7. URL 매핑

| URL | 페이지 |
|---|---|
| `/src/index.html` | 업체 정산 완료 내역 (목록) |
| `/src/pages/settlement-detail.html?company=&count=` | 정산상세 (정산 기간별 조회) |
| `/api/confirm-settlement?c=&r=&t=` | 정산서 확정 페이지 (HTML 응답) |
| `/api/*` (기타) | API 엔드포인트 (JSON 응답) |

## 8. 데이터 전달 방식

### 목록 → 상세

```js
var qs = 'pages/settlement-detail.html?company=' + encodeURIComponent(name) + '&count=' + count;
```

### 상세 → 메일 발송 모달

```js
MailSendModal.open({
  companyName: companyName,   // URL 파라미터에서 추출
  rowNo: rowNo,                // 현재 행 번호
  attachmentName: fileName,    // 자동 생성 또는 revFileName
  fetchAttachment: function() { ... },
  onSuccess: function(info) { recordMailSent(rowNo, info); }
});
```
