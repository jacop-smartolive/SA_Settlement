### 2-3. 검색 기능

```js
검색 타입: companyName / businessNo / ceo
필터링: DATA.filter(r => r[key].indexOf(kw) >= 0)
페이지 크기: 20 / 50 / 100
```

### 2-4. 최근 진행내역 컬럼

각 회사의 localStorage 3종 키를 읽어 **가장 최근 액션 1건**을 추출:

```js
function getCompanyActions(companyName) {
  // localStorage 3개 키 읽기:
  //   v11-detail-settle-{기업명}  (정산서 상태)
  //   v11-mail-{기업명}             (메일 발송)
  //   v11-dl-{기업명}              (5개 다운로드)
  // → 6개 작업유형별 최신 액션 수집
  // → 가장 최신 1건 = 셀에 표시
}
```

**셀 표시 (예시)**:
```
2026-04-21 11:13       ← 액션 컬러 (그레이/인디고/앰버/그린)
정산서 원본 다운로드   ← 밑줄, 클릭 가능
```

### 2-5. 진행내역 팝업

셀 클릭 시 **6개 작업유형 × 4컬럼** 표 모달:

| 유형 | 수정유무 | 진행내역 | 최근 일시 |
|---|---|---|---|
| 정산서 | 수정본 | 업로드 | 2026-04-21 11:33 |
| 가맹점 역발행 | 원본 | 다운로드 | 2026-04-21 11:38 |
| 가맹점 정발행 (일반) | 원본 | 다운로드 | 2026-04-20 16:10 |
| 가맹점 정발행 (간이) | - | - | - |
| 가맹점 대량이체 | 원본 | 다운로드 | 2026-04-21 11:45 |
| 회계전표 | 원본 | 다운로드 | 2026-04-20 16:31 |

- 정산서 행은 `확정` / `메일 발송` / `메일 재발송`도 표시
- 미진행 행은 `-` (회색)
- 일시 컬러 = 액션별 컬러 (B안 팔레트)

| 이용료(기업체) | 정수 | DYPNF Excel `G29` |

## 5. 서버 API

### 5-1. 엔드포인트 8개 (`server.js`)

| Method | Path | 설명 | 호출 위치 |
|---|---|---|---|
| POST | `/api/send-email` | Gmail SMTP 발송 + 정산서 확정 버튼 자동 삽입 | 메일 발송 모달 |
| POST | `/api/gmail-search` | IMAP 회신 검색 (from: 쿼리) | (예약, 미사용) |
| POST | `/api/gmail-detail` | 메일 상세 + 첨부 | (예약, 미사용) |
| POST | `/api/upload-revision` | 수정본 파일 서버 저장 | 수정본 첨부 시 |
| DELETE | `/api/upload-revision` | 수정본 파일 삭제 | 수정본 삭제 시 |
| GET | `/api/revision-file` | 수정본 다운로드 (메일 첨부용) | 메일 발송 시 |
| GET | `/api/confirm-settlement` | 메일 링크 클릭 → 확정 처리 (HMAC) | 수신자 메일 클릭 |
| GET | `/api/settlement-confirmations` | 회사별 확정 목록 (폴링) | 10초 폴링 |

### 5-2. `POST /api/send-email`

**입력 (FormData)**:
- `recipients` (JSON): `[{name, email}]`
- `subject`, `body`
- `companyName`, `rowNo` (정산서 확정 링크 생성용)
- `attachments`: 파일들 (multer)

**처리**:
1. SMTP 인증 (`smtp.gmail.com:465`)
2. 본문 끝에 정산서 확정 버튼 HTML/text 자동 삽입
3. HMAC 토큰 생성 → URL에 `t={token}` 포함
4. Nodemailer 발송

### 5-3. `GET /api/confirm-settlement`

**입력**: `c={기업명}` `r={행번호}` `t={HMAC토큰}`

**처리**:
1. HMAC 검증 (`verifyConfirmToken`, timing-safe)
2. 실패 → 403 인증 실패 페이지
3. 성공 → `confirmations.json`에 기록 + 그린 체크 카드 표시

### 5-4. `GET /api/settlement-confirmations?company=`

회사별 확정된 행 목록 반환:
```json
{
  "confirmations": {
    "77": { "datetime": "2026-05-11T14:32:00Z", "ip": "...", "userAgent": "..." }
  }
}
```

## 6. 메일 발송 & 정산서 확정 동기화

### 6-1. 메일 발송 흐름

```
[관리자] 정산상세 → 정산서 [발송 ▾] → 메일 발송
   ↓
MailSendModal 열림
   · 수신자/제목/본문 입력
   · 첨부 파일: revUploadedToServer ? 서버 수정본 : 디와이피엔에프 원본
   ↓
[클라이언트] FormData (recipients, subject, body, companyName, rowNo, attachments)
   ↓
[서버] POST /api/send-email
```
