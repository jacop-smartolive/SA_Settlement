## 1. 개요

**목적**: 정산 완료 내역 관리. 회사별 월간 정산서 다운로드/메일 발송/확정 처리.

**주요 차별점 (v2.0 대비)**
- v2.0의 디자인 그대로 이식하되 컬러 팔레트 B안 적용 (인디고 #4f46e5 / 앰버 #d97706)
- 모든 회사 상세 페이지가 디와이피엔에프 Excel 데이터로 통일
- 메일 본문에 정산서 확정 버튼 자동 삽입 + HMAC 토큰 검증
- 서버 폴링으로 자동 [완료] 동기화 (10초 주기)
- 수정본 파일을 서버에 업로드 → 메일 발송 시 첨부

## 6. 서버 API

### 6-1. 메일 발송 — `POST /api/send-email`

| 입력 | 설명 |
|---|---|
| recipients (JSON) | `[{name, email}]` |
| subject, body | 제목/본문 |
| companyName, rowNo | 정산서 확정 링크 생성용 |
| attachments | 파일들 (multer) |

**처리**:
1. Gmail SMTP 인증 (`smtp.gmail.com:465 SSL`)
2. 본문 끝에 정산서 확정 버튼 HTML/text 자동 삽입
3. HMAC 토큰 생성 (`signConfirmToken`)
4. URL: `/api/confirm-settlement?c=&r=&t={token}`

### 6-2. Gmail 회신 검색 — `POST /api/gmail-search`

- ImapFlow로 INBOX 접속
- `from:` 필터 또는 `X-GM-RAW` 쿼리
- 최근 50건 메타데이터 반환

### 6-3. 수정본 업로드 — `POST /api/upload-revision`

| 입력 | 설명 |
|---|---|
| companyName, rowNo | 식별자 |
| file (multipart) | .xlsx 파일 |

**저장 경로**: `revisions/{기업명}_{rowNo}.xlsx`

### 6-4. 정산서 확정 (메일 링크 클릭) — `GET /api/confirm-settlement`

| 입력 | 설명 |
|---|---|
| c | 기업명 (URL 인코딩) |
| r | 행 번호 (정수) |
| t | HMAC 토큰 (16자) |

**처리**:
1. `verifyConfirmToken()` 으로 토큰 검증 (timing-safe)
2. 실패 → 403 인증 실패 페이지
3. 성공 → `confirmations[c:r] = {datetime, ip, userAgent}` 저장
4. 사용자에게 성공 카드 표시

### 6-5. 확정 상태 폴링 — `GET /api/settlement-confirmations?company=`

회사별 모든 확정 행을 `{rowNo: {datetime}}` 형태로 반환.

## 8. 메일 발송 시스템

### 8-1. 발송 흐름

```
1. 정산서 [발송 ▾] → 메일 발송 클릭
   → MailSendModal.open() 호출
      · companyName, rowNo, attachmentName 전달
2. 사용자가 수신자/제목/본문 입력
3. fetchAttachment() 호출
   · st.revUploadedToServer = true → /api/revision-file
   · 아니면 → resources/samples/디와이피엔에프(주)_2026-04-30_정산서.xlsx
4. POST /api/send-email
   · 본문에 정산서 확정 버튼 HTML 자동 삽입
   · HMAC 토큰 자동 생성
5. Nodemailer로 Gmail SMTP 발송
6. recordMailSent() → localStorage 갱신
   · mail_sent: 시각 / mail_history: 발송 기록
```

## 9. 정산서 확정 동기화

### 9-1. 흐름

```
[발신 측 - 관리자]                     [수신 측 - 고객]
─────────────                         ─────────────
정산상세 페이지에서                     메일 수신
메일 발송 클릭                          ↓
  ↓                                    [✓ 정산서 확정 처리] 클릭
서버 → Gmail SMTP                      ↓
  ↓                                    http://host/api/confirm-settlement
메일 본문 자동 삽입:                     ?c=기업명&r=77&t={hmac}
  - HMAC 토큰 16자 포함                   ↓
                                       서버: verifyConfirmToken() 검증
                                       성공 → confirmations.json 기록
                                       성공 페이지 표시
                                       ↓
[관리자] 상세 페이지 열려 있음
   ↓
10초 폴링: /api/settlement-confirmations
   ↓
변경 감지 → rowStates[no].doneSettlement 갱신
   ↓
[완료 ▾] 버튼 (그린)으로 자동 표시
```

### 9-2. 폴링 절전 로직

```js

  document.hidden === true → stopPolling()

## 10. 수정본 처리

### 10-1. 첨부 흐름

```
[수정본 첨부] 메뉴 클릭
  ↓
File picker → 사용자가 .xlsx 선택
  ↓
1) 로컬 파싱 (parseRevisionFile)
   · '가맹점별 요약' 시트 (없으면 첫 시트) 사용
   · 라벨 정규식 검색:
     /^총계$/, /모바일\s*쿠폰\s*이용료\s*합계/, /식대\s*이용료\s*합계/, ...
   · 가맹점 명세: B/C/D/E/F/G 또는 A/B/C/D/E/F 컬럼 자동 감지
   · 결과 → st.revData
   · 실패 → DYPNF_APRIL_2026으로 fallback
  ↓
2) 서버 업로드 (POST /api/upload-revision)
   · revisions/{기업명}_{rowNo}.xlsx 저장
   · st.revUploadedToServer = true
  ↓
3) UI 갱신
   · 버튼 → [수정본 ▾] (인디고)
