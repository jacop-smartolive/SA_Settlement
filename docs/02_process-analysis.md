> v1.1 기준 — 정산 관리 업무 흐름 전체

```
┌──────────────────────────────────────────────────────┐

                  │ 회사 클릭
                  ↓
┌──────────────────────────────────────────────────────┐

                  ↓

└──────────────────────────────────────────────────────┘
```

## 2. 정산서 워크플로우 (행 단위)

### 상태 다이어그램

```
        ┌──────────────────────────────────────┐
        │  [원본 ▾] btn-outline (회색)           │ ← 초기 상태
        └──────────────────────────────────────┘
              │ 1) 원본 다운로드
              │    라벨: "원본 다운로드" 그레이
              │
              │ 2) 수정본 첨부 → 서버 업로드
              ↓
        ┌──────────────────────────────────────┐
        │  [수정본 ▾] btn-blue (인디고)          │
        └──────────────────────────────────────┘
              │ 3) 수정본 다운로드 / 파일변경 / 삭제
              │ 4) 원본 다운로드 (수정본 유지)
              │
              │ 5) 정산서 확정 (메뉴 클릭)
              │    또는 메일 수신자가 [확정 처리] 클릭
              ↓
        ┌──────────────────────────────────────┐
        │  [완료 ▾] btn-green (그린)             │
        │  라벨: "정산서 확정" 그린              │
        └──────────────────────────────────────┘
              │ 6) 확정 해제 (메뉴 클릭)
              │    → 이전 상태로 복귀
```

### 액션별 결과 매트릭스

| 상태 | 메뉴 | 결과 |
|---|---|---|
| **원본** | 원본 미리보기 | 모달 표시 |
| | 원본 다운로드 | .xlsx 다운로드 + 라벨 갱신 |
| | 수정본 첨부 | 파일 선택 → 파싱 → 서버 업로드 |
| | 정산서 확정 | doneSettlement 갱신 → 그린 [완료] |
| **수정본** | 수정본 미리보기 | 모달 (수정본 데이터) |
| | 수정본 다운로드 | 서버 파일 fetch + 다운로드 |
| | 수정본 파일변경 | 새 파일 선택 → 재파싱 |
| | 수정본 삭제 | 확인 후 클라이언트/서버 정리 |
| | 원본 미리보기/다운로드 | (위 동일) |
| | 정산서 확정 | (위 동일) |
| **완료** | 확정 해제 | doneSettlement = null → 이전 상태 |

## 3. 메일 발송 프로세스

```
[1] [발송 ▾] → 메일 발송 클릭
       ↓
[2] MailSendModal 열림
    - 수신자 추가/선택
    - 제목/본문 입력 (기본값 채워짐)
    - 추가 첨부 파일
       ↓
[3] "발송" 클릭
       ↓
[4] fetchAttachment:
    · revUploadedToServer = true → GET /api/revision-file
    · 아니면 → 디와이피엔에프 원본 Excel
       ↓
[5] POST /api/send-email
    · recipients, subject, body
    · companyName, rowNo
    · attachments (FormData)
       ↓
[6] 서버:
    · Gmail SMTP 인증
    · 본문에 정산서 확정 버튼 자동 삽입
    · HMAC 토큰 자동 생성 → URL 포함
    · Nodemailer 발송
       ↓
[7] 응답:
    · 클라이언트 mail_history 갱신
    · 버튼 [발송] → [재발송] (인디고)
    · 라벨 "메일 발송" (인디고)
```

## 4. 정산서 확정 동기화 프로세스

```
[발신측]                                  [수신측]
─────────                                ─────────
관리자 메일 발송
   ↓
[서버] HMAC 토큰 생성
   ↓
                                         수신자 메일 열기
                                            ↓
                                         [정산서 확정 처리] 클릭
                                            ↓
                                         GET /api/confirm-settlement
                                              ?c=&r=&t={token}
                                            ↓
[서버] verifyConfirmToken()
   ├ 검증 실패 → 403 에러 페이지
   └ 검증 성공 → confirmations.json 기록
                  ↓
                  성공 페이지 표시 ────→ [수신자에게 표시됨]

[관리자 화면]
   ↓
10초마다 GET /api/settlement-confirmations?company=
   ↓
변경 감지
   ↓
rowStates[no].doneSettlement = "YYYY-MM-DD|HH:MM"
   ↓
refreshRow(no) → [완료] 버튼 (그린)으로 자동 갱신
```

### 폴링 절전 모드

- `document.hidden = true` (탭 비활성) → `stopPolling()`
- `document.hidden = false` (탭 활성) → `startPolling()` 재시작
- `beforeunload` / `pagehide` → 영구 정리

## 5. 수정본 처리 프로세스

### 첨부 흐름

```
[수정본 첨부] 메뉴 클릭
   ↓
File picker (.xlsx, .xls, .csv)
   ↓
parseRevisionFile(file):
   1) SheetJS로 파싱
   2) '가맹점별 요약' 시트 (없으면 첫 시트)
   3) 라벨 정규식 검색:
      /^총계$/, /모바일\s*쿠폰/, /식대\s*이용료/, ...
   4) 가맹점 명세 자동 감지 (B/C/D/E/F/G 또는 A/B/C/D/E/F)
   5) 합계값 추출 → 계산값 자동 산출
   ↓
파싱 성공 → rowStates[no].revData 저장
파싱 실패 → DYPNF_APRIL_2026 fallback
   ↓
[병행] uploadRevisionToServer:
   POST /api/upload-revision → revisions/{기업명}_{rowNo}.xlsx
   ↓
UI 갱신:
   · 버튼 → [수정본 ▾] (인디고)
   · 라벨 → "수정본 업로드" (앰버)
   · 데이터 행 12개 셀 갱신
```

### 미리보기 분기

```
openSettlementPreview(no, isRev):
   ↓
revData 존재 + 데이터 있음?
   ├ Yes → useRich = true → 21개 가맹점 명세 + 합계 행 (모달 .wide)
   └ No  → useRich = false → 6열 요약 미리보기
```

## 7. 주요 시나리오 시퀀스

### 시나리오 A: 일반 정산 처리

```
1. 관리자: 회사 클릭 → 상세 진입
2. 관리자: 정산서 [원본] → 원본 다운로드
3. 관리자: 원본 검토 후 [원본] → 수정본 첨부
4. 관리자: 수정본 [수정본] → 수정본 미리보기 확인
5. 관리자: 메일 [발송] → 수신자에게 메일
6. 수신자: 메일 [정산서 확정 처리] 클릭
7. 시스템: 10초 내 자동 [완료] 표시
```

### 시나리오 B: 수정 → 재발송

```
1. 1차 메일 발송 후 데이터 오류 발견
2. 관리자: 수정본 [수정본] → 수정본 파일변경
3. 새 .xlsx 첨부 → 서버 업로드
4. 메일 [재발송] → 새 정산서 첨부됨
```

### 시나리오 C: 확정 해제

```
1. 잘못된 확정 처리
2. 관리자: [완료] → 확정 해제
3. 확인 다이얼로그 → 이전 상태 복귀
```
