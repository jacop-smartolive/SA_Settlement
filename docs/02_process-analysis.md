# 02. 프로세스 분석서

> v1.1에 구현된 **2개 화면**의 업무 흐름

## 1. 전체 흐름

```
┌────────────────────────────────────────────────────────┐
│ ① 업체 정산 완료 내역 (목록)                            │
│    /src/index.html                                     │
│    · 30개 기업 검색·페이징                              │
│    · 각 행: 회사명·완료건수·최근 진행내역               │
└──────────────────┬─────────────────────────────────────┘
                   │ 회사명 / 정산 완료 건수 클릭
                   ↓
┌────────────────────────────────────────────────────────┐
│ ② 정산상세 (정산 기간별 조회)                          │
│    /src/pages/settlement-detail.html?company=&count=   │
│    · 50개 행 (월별 정산 내역)                           │
│    · 각 행: 12개 데이터 + 9개 액션 컬럼                 │
└──────────────────┬─────────────────────────────────────┘
                   │ 행 단위 액션
                   ↓
            ┌──────┴──────────────┬──────────────┐
            ↓                     ↓              ↓
   [정산서 워크플로우]     [메일 발송]      [5종 다운로드]
    원본→수정본→완료     발송→재발송      가맹 역/정/지급/전표
                              │
                              ↓
                    [메일 본문 정산서 확정 버튼]
                              ↓
                    [수신자가 클릭]
                              ↓
                    [서버 기록 + 10초 폴링]
                              ↓
                    [관리자 화면 자동 [완료] 표시]
```

---

## 2. 화면 ①: 업체 정산 완료 내역

### 2-1. 주요 기능

| 기능 | 동작 |
|---|---|
| 검색 | 기업체명 / 사업자번호 / 대표자명 필터링 |
| 페이징 | 20 / 50 / 100건 단위 |
| 회사 클릭 | 정산상세 페이지로 이동 |
| 정산 완료 건수 클릭 | 정산상세 페이지로 이동 |
| **최근 진행내역 셀 클릭** | 6개 작업유형 × 4컬럼 팝업 |

### 2-2. 최근 진행내역 수집 로직

```
각 회사마다 localStorage 3개 키 읽기:
  · v11-detail-settle-{기업명}  → 정산서 lastAction, doneSettlement
  · v11-mail-{기업명}            → mail_sent (+ mail_history로 재발송 판정)
  · v11-dl-{기업명}              → 5개 다운로드 키
       ↓
6개 작업유형별 최신 액션 추출 (시간 기준 latest)
       ↓
가장 최신 1건 → 셀에 표시
       ↓
셀 클릭 → 6개 유형 모두 팝업 표시
```

### 2-3. 진행내역 팝업

| 유형 | 수정유무 | 진행내역 | 최근 일시 |
|---|---|---|---|
| 정산서 | 원본/수정본/- | 다운로드/업로드/확정/메일 발송/재발송 | YYYY-MM-DD HH:MM |
| 가맹점 역발행 | 원본 | 다운로드 | ... |
| 가맹점 정발행 (일반) | 원본 | 다운로드 | ... |
| 가맹점 정발행 (간이) | 원본 | 다운로드 | ... |
| 가맹점 대량이체 | 원본 | 다운로드 | ... |
| 회계전표 | 원본 | 다운로드 | ... |

- 작업 이력이 없는 행은 `-` (회색)
- 일시 컬러 = 액션별 컬러 (B안 팔레트)

## 3. 화면 ②: 정산상세 (정산 기간별 조회)

### 3-1. 페이지 구성

```
[페이지 위치] 정산 관리 > 정산 완료 내역 > 정산 기간별 조회
[h2] {기업명}
결과: {정산건수}

[표 21열, 가로 스크롤]
  ┌────────────── 데이터 12열 ─────────────┬─── 액션 9열 ───┐
  │ 번호 / 정산기간 / 결제건수 / 사용포인트 │ 정산서 / 메일발송│
  │ / 요금단가 / 이용료(기업체) / ...      │ / 가맹 역/정/간   │
  │ / 가맹점 이체 제외 금액                 │ / 대량이체 / 회계 │
  │                                        │ 전표 / 삭제 / 요청일│
  └────────────────────────────────────────┴────────────────┘
```

### 3-2. 데이터 컬럼 (12개)

모든 회사가 디와이피엔에프 Excel 데이터를 동일하게 사용:

- **최신 행 (no = 정산건수)**: Excel 값 그대로
- **이전 행**: 시드 기반 ±15% 변동 (월별 시뮬레이션)

### 3-3. 액션 컬럼 - 정산서 워크플로우

```
        ┌────────────────────────┐
        │ [원본 ▾] btn-outline   │ ← 초기 상태
        └──────────┬─────────────┘
                   │ 수정본 첨부
                   ↓
        ┌────────────────────────┐
        │ [수정본 ▾] btn-blue    │   인디고 #4f46e5
        └──────────┬─────────────┘
                   │ 정산서 확정 메뉴 클릭
                   │ OR 메일 수신자 [확정 처리] 클릭
                   ↓
        ┌────────────────────────┐
        │ [완료 ▾] btn-green     │   그린 #059669
        └────────────────────────┘
```

### 3-4. 정산서 메뉴 항목

| 상태 | 메뉴 |
|---|---|
| 원본 | 원본 미리보기 / 원본 다운로드 / 수정본 첨부 / ─ / 정산서 확정 |
| 수정본 | 수정본 미리보기 / 다운로드 / 파일변경 / 삭제 / ─ / 원본 미리보기 / 다운로드 / ─ / 정산서 확정 |
| 완료 | (위 동일) / ─ / 확정 해제 |

### 3-5. 메일 발송 컬럼

| 상태 | 버튼 | 메뉴 |
|---|---|---|
| 미발송 | `[발송 ▾]` (회색 아웃라인) | 메일 발송 / 발송 내역 |
| 발송됨 | `[재발송 ▾]` (인디고) | 메일 재발송 / 발송 내역 |

메일 발송 시 본문에 **정산서 확정 그린 버튼** 자동 삽입 (HMAC 토큰 포함).

### 3-6. 5개 다운로드 전용 컬럼

가맹 역발행 / 정발행 (일) / 정발행 (간) / 대량이체 / 회계전표:
- 단일 `[다운로드]` 버튼 (드롭다운 없음)
- 클릭 시 .xlsx 다운로드 + 일시 라벨 갱신 (그레이)

## 4. 핵심 시나리오

### 시나리오 A: 일반 정산 처리

```
1. 관리자: 목록에서 회사 클릭 → 상세 진입
2. 정산서 [원본 ▾] → 원본 다운로드 (검토)
3. 수정본 필요 시: [원본 ▾] → 수정본 첨부 → 새 .xlsx 선택
   → 서버 업로드 + 파싱 + UI 갱신 ([수정본 ▾] 인디고)
4. [수정본 ▾] → 수정본 미리보기 (Excel 명세 모달)
5. 메일 [발송 ▾] → 메일 발송 → 수신자 입력 → 발송
6. 수신자: 메일 본문 [정산서 확정 처리] 클릭
7. 서버: HMAC 검증 → 확정 기록
8. 관리자 화면 (10초 폴링) → [완료 ▾] 그린 자동 표시
```

### 시나리오 B: 수정본 재발송

```
1. 1차 메일 발송 후 데이터 오류 발견
2. 정산서 [수정본 ▾] → 수정본 파일변경 → 새 파일
3. 메일 [재발송 ▾] → 메일 재발송 → 새 정산서 첨부
```

### 시나리오 C: 확정 해제

```
1. 잘못된 확정 처리
2. [완료 ▾] → 확정 해제 → confirm 다이얼로그
3. doneSettlement = null → 이전 상태로 복귀
```

### 시나리오 D: 5종 다운로드

```
1. 가맹 역발행 [다운로드] 클릭 → .xlsx 다운로드 + 라벨 갱신
2. 일시 라벨: "YYYY-MM-DD HH:MM / 원본 다운로드" (그레이)
3. 같은 행 다른 컬럼도 동일하게 동작
```

```
┌─ 브라우저 ───────────────────────────────────────────┐

│       │                          │                   │

│       ↓                          ↓                   │
│  ┌─────────────────────────────────────┐            │
│  │ v11-detail-settle-{기업명}           │            │
│  │ v11-mail-{기업명}                    │            │
│  │ v11-dl-{기업명}                      │            │

                   │ fetch
                   ↓
┌─ 서버 (server.js) ─────────────────────────────────┐

│                                                    │
│  파일:                                              │
│  · confirmations.json   (확정 영속)                 │

│                                                    │

```
